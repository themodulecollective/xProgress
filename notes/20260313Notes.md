## 0.1.0 Release — Bugs Found and Fixed

### Critical
- `FunctionsToExport` in xProgress.psd1 was missing `Start-xProgress`,
  `Suspend-xProgress`, `Resume-xProgress` — these would not be available
  to module consumers after publishing

### Code bugs (xProgress.psm1)
- `-Id` parameter in `New-xProgress` declared but never applied to the
  instance object. Fixed: now uses provided Id when present.
- Division by zero in `Write-xProgress` when `-DoNotIncrement` is used
  as the first call (Counter=0). `0 % interval = 0` triggered the display
  block, then `elapsedSeconds / 0` threw. Fixed: added `$counter -gt 0`
  guard on the display block condition.
- Auto-generated `CurrentOperation` text: `$progressItem` was not capped,
  so batch end count could exceed total (e.g. "101 through 110 of 105").
  Fixed: `[Math]::Min($counter + $progressInterval - 1, $xPi.total)`

### Manifest updates (xProgress.psd1)
- FunctionsToExport: added 3 timer functions
- Description: updated to reflect full 0.1.0 scope
- ReleaseNotes: populated from commented-out state

## Pester Feasibility Evaluation (Future Session)

**Verdict: Feasible, no module changes required.**

### Key techniques
- Mock `Write-Progress` to test call frequency and parameters without
  capturing stream 5
- Use `Get-xProgress` to inspect all module state (ProgressTracker is
  accessible through the public function)
- Reset module state between tests via Remove-Module / Import-Module
  in BeforeEach — avoids polluting production code with test helpers

### High-value test cases identified
- Interval calculation for all CalculatedProgressInterval values
- Write-Progress called exactly once on first item regardless of interval
- Write-Progress call count matches expected interval across N items
- -DoNotIncrement: counter does not advance; no division by zero on first call
- $progressItem never exceeds Total in CurrentOperation text (regression)
- -Id parameter applied to instance when provided (regression)
- Complete-xProgress removes instance from tracker and calls with -Completed
- Stopwatch IsRunning state after Start / Suspend / Resume calls

### Harder to test
- Exact SecondsRemaining values (depend on real elapsed time) — assert
  parameter was passed, not specific value
- Write-Information output from Complete-xProgress — capturable via
  -InformationVariable or 6>&1 but verbose to set up
